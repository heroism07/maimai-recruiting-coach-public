# AI 接管默认执行说明（脉脉）

## 适用场景

当任务涉及脉脉页面操作时，默认使用 AI 接管。脚本仅做结构化与同步，不做主链路页面点击。

## 标准执行顺序

1. 优先接管已打开的“人才银行/搜索页”分页（先 `tab-list` 再 `tab-select`）。
2. 确认当前页仍是用户已配置筛选条件的页面（先 `snapshot` 校验筛选区）。
3. 仅在会话内找不到可接管分页时，才 `state-load + goto https://maimai.cn/ent/v41/index`。
4. 检查登录态，未登录时等待用户登录。
5. 登录后进入“招人 -> 搜索”。
6. 按职位模板应用筛选条件。
7. 筛选条件设置后必须点击“搜索”并确认结果已刷新（该项属于流程动作，不属于六项检查）。
8. 按分页执行候选人评估：
   - 每页必须按卡片顺序逐一判定，禁止整页浏览后再回看。
   - 卡片不匹配直接跳过。
   - 可沟通或储备观察进入详情；有附件则评估附件，并在该候选人评估当下完成附件处理。
   - 详情评估按模板三维规则执行（`industry_core_background`、`party_a_core_background`、`domain_relevance`）。
9. 每位候选人评估完成后写入字段 `六项检查结果`（取值仅允许 `通过` / `不通过`）。
   - `六项检查结果=通过` 仅表示六项都已判断且有依据，不表示候选人质量通过。
   - 单项可为负向判断（如低意向），仍计入“已判断”。
   - 无证据时必须标记“未判断”，不得编造事实补齐六项。
10. 每页结束仅汇总候选人，不执行飞书同步。
11. 翻下一页继续，直到当前模板全部页面完成。
12. 当前模板全部页面完成后，一次性同步白名单候选人（`可沟通`、`储备观察`，含附件简历）到飞书；且仅同步 `六项检查结果=通过` 的记录。
13. 若中途暂停/异常未完成模板，默认不自动同步，等待人工确认。

## 关键交互细则

1. `职位名称`、`就职公司` 等联想控件必须“输入 -> 点击候选项”，仅输入文本不算生效。
2. 多值筛选要逐个“输入 -> 点击候选项”，每一项点中后再录入下一项。
3. 禁止用 Enter/Tab/失焦作为联想控件选中兜底；未点击候选项一律视为失败。
4. 页面动作保持拟人化节奏，避免高频连续操作。
5. `行业方向`、`年龄` 这类弹层控件必须点击“确定”才算提交。
6. `职位名称`需按业务粒度点选，若要求 `目标岗位负责人` 与 `目标岗位总监` 独立，不得用合并候选项替代。
7. `就职公司`优先主公司名称，禁止误选营业部/分公司作为模板默认项。

详细字段级规则见：`references/filter-ui-operation-rules.md`

## 接管与人工切换条件

以下场景必须转人工：
1. 验证码、访问受限、风控提示。
2. 登录态异常且短时间内无法恢复。
3. 关键筛选字段无法确认已生效。
4. 页面结构突变导致流程连续失败。

## 运行隔离要求

1. 仅允许自动化 `*.maimai.cn`。
2. 使用独立浏览器目录：`data/maimai-browser-profile`。
3. 禁止干扰用户日常浏览器会话。
4. 同一轮接管必须使用同一浏览器会话，禁止中途切换到另一套控制通道。


## 2026-03-25 执行补充（校正版）

1. 筛选条件设置完成后必须点击“搜索”按钮，确认结果已刷新后再进入候选人评估（流程动作，不属于六项检查）。  
2. 候选人评估必须逐卡片推进，禁止整页浏览后回看。  
3. 误触候选人详情页时，先关闭详情抽屉并回到当前卡片序位，再继续评估。  
4. 详情或附件未复核完成前，不得给出可沟通/储备观察结论。  
5. 每位候选人必须写入 `六项检查结果`，取值仅允许 `通过` / `不通过`。  
6. 若 `六项检查结果=不通过`，仅回写人工复核队列，不触发同步。

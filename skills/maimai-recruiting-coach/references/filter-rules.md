# 人才筛选规则（岗位+场景）

## 1. 执行顺序（强约束）

1. 卡片初判：明显不匹配直接跳过。  
2. 详情复核：仅对“可沟通/不确定”进入详情评估。  
3. 附件复核：有附件简历时必须结合附件再给结论。  
4. 判定顺序固定：每页按卡片顺序逐一评估，禁止整页浏览后回看。

## 2. 三维评估模型（模板复用）

详情评估统一使用三维：

1. `industry_core_background`：行业核心背景（strong/medium/weak）
2. `party_a_core_background`：甲方核心岗位负责人背景（strong/medium/weak）
3. `domain_relevance`：科技互联网相关性（high/medium/low）

等级映射分值：

1. `industry_core_background`: strong=100, medium=70, weak=30
2. `party_a_core_background`: strong=100, medium=65, weak=20
3. `domain_relevance`: high=100, medium=70, low=35

加权总分（0-100）：

`score = 0.35*ib + 0.45*party_a + 0.20*tech`

## 3. 结论阈值（严格版）

1. `score >= 85` -> `可沟通`
2. `70 <= score <= 84` -> `储备观察`
3. `score < 70` -> `不合适`

## 4. 硬排除项（Hard Reject）

命中任一项直接 `不合适`：

1. 明确求职方向长期偏离目标岗位负责人主线（仅投资/IR/顾问导向），且无可验证的核心岗位一号位经历。
2. 关键经历无法在详情或附件中完成基本核验（信息缺失且无法复核）。

## 5. 模板字段落位（必须）

为保证后续复用一致，规则必须写入模板：

1. `筛选策略`：写人读版规则（维度定义、权重、阈值、硬排除、判定顺序）。
2. `筛选条件JSON`：写机读版规则，建议包含 `detail_evaluation_rule`。

## 6. 输出建议

对每位候选人输出：

1. `match_decision`：`contact` / `hold` / `reject`
2. `reason`：命中的关键依据
3. `confidence`：0-1


## 2026-03-25 评估与硬排除补充（校正版）

1. 硬排除新增：主求职方向长期偏离关键岗位主线（翻译/算法/产品运营/柜员等）且无可验证核心岗位一号位经历时，直接判定不合适。  
2. 三维评分仅用于匹配强度量化，不得覆盖硬排除结论。  
3. 判定顺序固定：卡片初判 -> 详情复核 -> 附件复核 -> 三维打分 -> 阈值映射 -> 硬排除复判。  
4. 阈值强约束：score >= 85 为可沟通；70-84 为储备观察；< 70 为不合适。  
5. score 与 conclusion 必须一致，不一致时禁止同步。  
6. score/conclusion 校验用于业务结论一致性，不等同于六项完整性校验。  
7. 六项完整性校验仅判断“六项是否都已判断且有依据”。

# 飞书筛选模板 V2 字段设计（脉脉双向同步）

目标：
1. 在飞书多维表格配置模板记录后，可完整还原到脉脉筛选。
2. 在脉脉已设置好的筛选条件，可回写成结构化字段，支持二次复用。

## 1. 字段分层

### 1.1 兼容层（旧字段，保留）
- `城市地区`
- `学历要求`
- `工作年限`
- `就职公司`
- `年龄`
- `期望月薪`
- `智能筛选_*`（旧布尔字段）

用途：
- 兼容旧记录与旧脚本。
- 做人眼快速浏览摘要。

### 1.2 结构化层（V2 新增）
- `模版基础名`：模板归组索引
- `模版版本`：模板版本号（整数）
- `父记录ID`：上一版本 record_id
- `变更摘要`：本次修改说明
- `职位需求`：用户原始职位要求（画像输入）
- `筛选策略`：规则/LLM 拆解后的筛选策略说明（需包含详情评估三维规则与阈值）
- `筛选模式`：`结构化模式(新字段优先)` / `兼容模式(旧字段)`
- `筛选条件JSON`：保留原始快照（便于排查和回溯），并承载结构化评估规则（`detail_evaluation_rule`）
- `招呼语要求`：用于约束该模板下的招呼语生成边界（如“保密招聘，禁止出现岗位关键词”）

#### 城市

#### 年龄

- `年龄_最高`（数字）

#### 期望月薪

#### 就职公司

- `就职公司_列表`（文本，多值建议 `;` 分隔）

#### 智能筛选

- `智能筛选_公开求职意向_状态`（多选）：`正在看机会`、`关注行情`、`半年不看机会`
- `智能筛选_近期有动向_开关`（单选）：`默认(不筛选)` / `启用`
- `智能筛选_近期有动向_范围`（单选）：`近7天`、`近14天`、`近1个月`、`近3个月`
- `智能筛选_有附件简历_开关`（单选）：`默认(不筛选)` / `启用`
- `智能筛选_有过意向_开关`（单选）：`默认(不筛选)` / `启用`
- `智能筛选_企业号互动_开关`（单选）：`默认(不筛选)` / `启用`
- `智能筛选_企业号互动_类型`（多选）：`企业号粉丝`、`企业职位互动`、`企业动态互动`、`精准营销推广`

## 2. 读写优先级

### 飞书 -> 脉脉
1. 若按基础名拉取模板，默认读取该基础名最新版本。
2. 若 `筛选模式=结构化模式(新字段优先)`，优先读取 V2 结构化字段。
3. 若 V2 字段为空，则回退到旧字段。
4. 执行落地时，区间字段优先走“自定义范围”入口。

### 脉脉 -> 飞书
1. 回写 V2 结构化字段（精确表达区间/状态）。
2. 默认不再写入冗余旧字段；如需临时回滚，可显式开启 `--write-legacy-fields true`。
3. 同步 `筛选条件JSON` 作为合规检查快照。
4. 差异检测有变化时新增版本（`模版基础名@vNNN`），无变化则跳过。

## 3. 详情评估规则（模板复用约定）

1. 模板必须沉淀三维评估：`industry_core_background`、`party_a_core_background`、`domain_relevance`。
2. `筛选策略`存放人读规则（维度解释、阈值、硬排除、判定顺序）。
3. `筛选条件JSON`建议存放机读规则，建议结构：
   - `detail_evaluation_rule.dimensions`
   - `detail_evaluation_rule.weights`
   - `detail_evaluation_rule.thresholds`
   - `detail_evaluation_rule.hard_reject`
   - `detail_evaluation_rule.decision_flow`
4. 执行评估时优先读取 `筛选条件JSON.detail_evaluation_rule`；缺失时回退 `筛选策略`文本规则。

## 4. 关键约定

1. 复选框类条件（如 `有附件简历`）的“未勾选”统一表示 `默认(不筛选)`，不等于“筛选为否”。
2. `就职公司_列表` 建议存公司名或公司标签，值用 `;` 分隔。
3. `城市地区_列表` 建议存城市名，值用 `;` 分隔。
4. `期望月薪_*` 单位统一使用 `K`（数值字段仅存数字）。


## 2026-03-25 detail_evaluation_rule 补充（校正版）

建议在 `detail_evaluation_rule` 中明确以下约束（或等价语义）：

1. `enforce: true`：阈值与硬排除按强约束执行。  
2. `hard_reject` 必须包含“主求职方向偏离岗位主线”。  
3. `decision_flow` 必须包含：逐卡片评估、禁止整页回看、附件当下下载、模板完成后批量同步。  
4. `thresholds` 建议完整包含：`contact_min`、`hold_min`、`hold_max`。

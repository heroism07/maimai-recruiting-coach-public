# 脉脉筛选控件操作规范（AI 探索版）

更新时间：2026-03-24（含当日探索实测）

## 适用范围

1. 页面：`https://maimai.cn/ent/v41/recruit/talents?tab=1`（人才银行-搜索）。
2. 目标：把模板条件稳定落到页面筛选栏，并可回读验证。
3. 执行方式：AI 接管可见窗口（headed），单会话单通道执行。

## 探索模式硬约束（防误触候选人详情）

1. 探索阶段只允许点击“搜索条件”区域与“左侧从结果中筛选”区域，禁止点击候选人列表卡片。
2. 每次操作前先确认目标是筛选控件标题文本，再执行点击。
3. 若出现右侧候选人详情抽屉（`ant-drawer`），先 `Esc` 或点关闭，再继续筛选。
4. 联想控件必须执行“输入 -> 点击候选项”，禁止用 Enter/Tab/失焦兜底选中。
5. 每个筛选项设置后都要回读摘要文本；未回读视为未生效。

## 通用规则（所有筛选项）

1. 先校验页面在“搜索条件”区域，再执行筛选项操作。
2. 多值字段逐项处理，避免一次性粘贴导致未命中候选。
3. 若候选项出现歧义（总公司 vs 分公司/营业部），优先主实体；无法确认则转人工。
4. 弹层/面板类控件，只有点击“确定”才算提交。

## 逐筛选框可行方案（实测）

| 筛选框 | 控件类型 | 一条可行操作路径 | 提交方式 | 回读信号 |
|---|---|---|---|---|
| 关键词 | 文本输入 | 输入关键词 -> 点击“搜索”按钮 | 按钮提交 | 输入框保留关键词、结果数变化 |
| 关键词逻辑（所有/任一） | 下拉单选 | 点击“任一/所有” -> 选目标项 | 即时 | “满足 X 关键词”显示目标值 |
| 城市地区 | 面板多选 | 点击城市地区 -> 勾口径（期望/现居）-> 点城市项 | 即时 | 摘要显示 `城市地区：期望/现居 - ...` |
| 学历要求 | 快捷项 + 自定义下拉 | 点击学历要求 -> 选快捷项，或设置最低/最高学历 | 点“确定”（自定义时） | 摘要出现学历条件 |
| 工作年限 | 快捷项 + 自定义下拉 | 点击工作年限 -> 选区间，或设置最低/最高年限 | 点“确定”（自定义时） | 摘要出现年限条件 |
| 就职公司 | 联想多值 | 点击就职公司 -> 确认范围（正任职/曾任职）-> 对每家公司执行“输入 -> 点击候选项” | 即时 | 摘要出现公司清单标签 |

| 行业方向 | 弹层树状选择 | 点击行业方向 -> 选行业 -> 点“确定” | 弹层“确定” | 摘要显示 `行业方向：...` |
| 毕业学校 | 输入/推荐项 | 点击毕业学校 -> 输入学校 -> 点击候选项（或推荐项） | 即时 | 摘要出现毕业学校条件 |
| 专业 | 输入联想 | 点击专业 -> 输入专业 -> 点击候选项 | 即时 | 摘要出现专业条件 |
| 性别 | 面板单选 | 点击性别 -> 选“不限/男/女” | 即时 | 摘要出现性别条件（非不限时） |
| 年龄 | 自定义下拉 + 面板确认 | 点击年龄 -> 选最低/最高年龄候选项（最终必须点目标年龄项）-> 点“确定” | 面板“确定” | 摘要出现年龄边界 |
| 期望月薪 | 自定义下拉 + 面板确认 | 点击期望月薪 -> 选最低/最高月薪 -> 点“确定” | 面板“确定” | 摘要出现月薪区间 |
| 家乡 | 输入/列表选择 | 点击家乡 -> 输入或点城市列表项 | 即时 | 摘要出现家乡条件 |

| 智能筛选：有附件简历 | 复选 | 勾选“有附件简历” | 即时 | 勾选状态变化、结果数变化 |
| 智能筛选：有过意向 | 复选 | 勾选“有过意向” | 即时 | 勾选状态变化、结果数变化 |
| 智能筛选：企业号互动 | 复选（可带类型） | 勾选“企业号互动” -> 需要时选互动类型 | 即时 | 勾选状态变化、结果数变化 |

## 关键防错规则（重点）

1. `职位名称`、`就职公司`严格逐项“输入 -> 点击候选项”；每点一项就回读一次。
2. `职位名称`若要求独立三项（`目标岗位负责人`、`目标岗位总监`、`目标岗位负责人`），不得用“目标岗位总监”合并项替代。
3. `就职公司`优先主实体，避免误选营业部/分支机构。
4. `年龄`、`期望月薪`不要只靠键盘方向键计数；最终必须点击可见目标候选项，再点“确定”。
5. 出现详情抽屉遮挡时，先关闭抽屉，再进行筛选提交。

## 执行验收清单（最小）

1. 关键词与关键词逻辑已生效。
2. 城市、职位名称、就职公司已生效，且联想项全部来自“输入 -> 点击候选项”。
3. 行业方向、年龄、期望月薪这三类“需确认”控件已点击“确定”。
4. 智能筛选若有开启，勾选状态与目标配置一致。
5. 无误触候选人详情页；若误触，已先关闭抽屉再继续筛选。


## 2026-03-25 筛选生效补充（校正版）

1. 任一筛选条件更新后，必须点击“搜索”按钮才视为生效。  
2. 点击“搜索”后需执行回读校验（筛选摘要/结果数量变化）；未通过回读不得进入候选人评估。  
3. 若用户已手动点击“搜索”，AI 需先确认页面结果已刷新，再继续后续流程。

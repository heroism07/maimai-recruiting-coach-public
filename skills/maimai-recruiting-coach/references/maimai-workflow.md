# 脉脉招聘执行闭环（业务版）

## 目标

把“岗位画像 -> 条件筛选 -> 候选人评估 -> 飞书同步 -> 模板沉淀”做成可重复执行的标准流程。

## 业务闭环

1. 用户确认岗位要求与优先级（必须项、加分项、排除项）。
2. 生成或复用职位模板（飞书版本化保存）。
3. AI 接管脉脉按模板筛选，筛选条件设置后必须点击“搜索”并确认结果刷新。
4. AI 逐页评估候选人，并在每位候选人评估完成后写入 `六项检查结果`（`通过` / `不通过`）。
5. 每页结束仅汇总评估结果，不立即同步飞书。
6. 模板全部页面评估完成后，一次性同步飞书候选人表（仅 `可沟通`、`储备观察`，且 `六项检查结果=通过`，含附件）。
7. 模板结束后回写结果摘要与优化建议。
8. 若模板有改动，回写新版本供下一轮复用。

## 单模板执行标准

1. 先锁定本轮模板版本，再开始筛选，避免中途换标准。
2. 页面评估顺序固定：卡片初判 -> 详情复核 -> 附件复核（如有）。
3. 每页必须按卡片顺序逐一判定，禁止整页浏览后再回看。
4. 六项业务检查固定为：基础信息完整、硬性条件匹配、相关经验对口、职业稳定性可接受、求职意愿与沟通可达性明确、风险备注完整。
5. 六项检查结果字段固定为 `六项检查结果`，取值仅允许 `通过` / `不通过`。
6. 六项检查结果=通过，仅表示六项都已判断且有依据，不表示候选人质量通过。
7. 单项负向判断（如“低意向”“不匹配”）仍属于“已判断”，不会单独导致完整性不通过。
8. 详情复核必须执行模板三维评估（`industry_core_background`、`party_a_core_background`、`domain_relevance`）与阈值判定。
9. 不合适候选人直接跳过，不入飞书候选人表。
10. 模板内分页必须闭环跑完或明确人工暂停点；未跑完默认不自动同步。

## 多模板批处理标准

1. 先确认模板清单再启动任务。
2. 采用串行执行，模板间结果独立记录。
3. 任一模板触发暂停条件时，整个会话可暂停并等待人工接管。

## 输出物

1. 候选人表模板级一次性写入结果。
2. 模板执行结果（状态、人数、错误摘要、建议）。
3. 会话汇总文件（用于复盘和下轮优化）。


## 2026-03-25 流程补充（校正版）

1. 在脉脉侧完成筛选条件设置后，必须点击“搜索”按钮才可进入候选人评估。  
2. 候选人同步属于模板收口动作，不允许在分页评估过程中穿插同步。  
3. 评估记录必须可追溯：页码、卡片序号、详情复核状态、附件处理时点、六项检查结果。  
4. 模板内出现缺页或中断时，默认不执行候选人同步。  
5. 若六项检查结果=不通过，必须回写未判断项、依据缺失原因与处理状态（待人工复核），不得进入同步。

# 飞书字段同步规则（最新版）

更新时间：2026-03-24

## 0. 筛选模板索引与版本

- 筛选模板主索引字段为 `模版名称`（兼容 `模板名称`、`场景名称` 历史字段）。
- `职位需求` 用于存放用户岗位画像原文；`筛选策略` 用于存放系统拆解后的筛选方案说明。
- `筛选策略` 必须包含详情评估三维规则（`industry_core_background`、`party_a_core_background`、`domain_relevance`）与结论阈值。
- `筛选条件JSON` 建议同步存储结构化评估规则（`detail_evaluation_rule`），用于下次模板复用时保持一致判定维度。
- 新增版本命名规则：`模版名称 = 模版基础名@vNNN`。
- 输入基础名拉取时默认命中最新版本；输入完整版本名时精确命中。
- 双向同步筛选条件时默认采用 append-only：
  - 条件有变化：新增记录；
  - 条件无变化：跳过写入；
  - 默认不覆盖原记录。
- 仅在显式指定 `update-existing` 时允许覆盖更新。
- 冗余字段瘦身策略固定为：
  - 先停写：默认不再写入冗余旧字段（可用 `--write-legacy-fields true` 临时回滚）；
  - 再迁移：执行 `upgrade-filter-table-v2.js` 回填结构化字段；
  - 最后删除：执行 `prune-filter-table-fields.js` 删除冗余旧字段。

## 1. 主键与姓名

- 姓名只写入主列 `候选人`。
- `候选人姓名` 视为废弃字段，不再写入、不再创建。

## 2. 单选字段（必须）

以下字段统一为单选类型（`SingleSelect`）：

- `有无附件简历`：`是` / `否` / `未知`
- `详情页已复核`：`是` / `否` / `未知`
- `附件简历已查看`：`是` / `否` / `未知`
- `结论`：`可沟通` / `储备观察` / `不合适`
- `六项检查结果`：`通过` / `不通过`
- 字段语义补充：`六项检查结果` 是完整性字段（是否完成六项判断），不是质量字段。

## 3. 状态与学历写法

- `状态情况` 直接写完整信息，优先写“近期有动向”的动态详情。
  - 示例：`正在看机会；近期有动向（更改求职状态为正在看机会 3月5日；活跃值提升 2月2日）`
- `学历情况` 必须包含学校 + 学历 + 在校时间（已合并写入）。

## 4. 附件写法

- 不再使用 `附件简历预览链接`。
- 附件写入 `附件简历附件`（附件类型字段）。
- 上传文件名统一为：`候选人姓名.pdf`。
- 非 PDF 附件跳过上传并记录失败原因。

## 5. 标签与话术

- 职业标签写入 `职业标签` 字段（`标签` 字段已废弃）。
- `打招呼话术草稿` 必须保留，用于人工确认后再发送。
- 自动话术生成默认仅对 `结论=可沟通` 生效，并受模板 `招呼语要求` 约束。

## 6. 导入范围

- 同步范围固定为：`结论=可沟通`、`结论=储备观察`。
- 同步准入固定为双门槛：`六项检查结果=通过` 且 `结论=可沟通/储备观察`。
- `结论=不合适` 默认不入库（可在原始评估文件保留用于复盘）。
- 同步时机改为模板级批量同步：当前模板全部页面评估完成后，一次性入库。
- 若模板中途暂停/异常未完成，默认不自动同步，待人工确认后再执行。
- 若 `六项检查结果=不通过`，必须回写未判断项与依据缺失原因，并标记 `待人工复核`，禁止同步。

## 7. 候选人字段最小必填

同步前必须确保以下字段可用：
- `候选人`
- `结论`
- `状态情况`
- `学历情况`
- `详情页已复核`
- `附件简历已查看`
- `六项检查结果`

当字段缺失时：
1. 标记待补全并进入人工复核。
2. 不允许静默写入空值冒充有效记录。


## 2026-03-25 同步机制补充（校正版）

1. 同步白名单固定为：可沟通、储备观察；不合适禁止入库。  
2. 同步时机固定为模板级批量同步：该筛选条件下全部页面评估完成后一次性同步。  
3. 可沟通/储备观察且有附件简历时，评估当下即下载并记录；同步阶段仅上传，不再二次抓取。  
4. 同步前必须通过双门槛：`六项检查结果=通过` 且 `结论=可沟通/储备观察`。  
5. 字段摘要样例对齐：候选人“毛玉娇”使用“公司｜职位｜任职时间”三元组写法。
